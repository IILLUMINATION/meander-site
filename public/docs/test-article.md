# Создание первого квеста

> Полное руководство по созданию вашего первого текстового квеста в Meander

## Введение

В этом руководстве мы создадим простой текстовый квест с ветвящимся сюжетом, изображениями и музыкой. Если вы ещё не установили Meander — сделайте это в разделе [Скачать](/#download).

---

## Шаг 1: Создание проекта

Откройте Meander и нажмите **«Новый квест"**. Появится окно с настройками:

- **Название** — имя вашего квеста
- **Автор** — ваш ник или псевдоним
- **Описание** — краткое содержание для маркета
- **Обложка** — изображение для превью

![Создание нового проекта](/images/мокап программы из гугл плей красивый(не реальный скрин).jpg)

*Окно создания нового проекта*

---

## Шаг 2: Первый узел

Каждый квест состоит из **узлов** — это отдельные сцены с текстом, изображениями и вариантами выбора.

### Добавление текста

Напишите текст для первой сцены. Это может быть описание обстановки или диалог персонажа:

> Вы стоите перед старым замком. Ворота приоткрыты, изнутри доносится тихая музыка. Что вы сделаете?

### Добавление изображения

Добавьте фоновое изображение для атмосферы. Поддерживаются форматы **JPG**, **PNG** и **WebP**.

![Замок — пример фона](/images/мокап программы из гугл плей красивый(не реальный скрин) 2.jpg)

*Фоновое изображение замка*

## Шаг 3: Создание ветвлений

Добавьте варианты выбора, чтобы создать ветвящийся сюжет:

1. **Войти в замок** → ведёт к узлу «Внутри замка»
2. **Обойти вокруг** → ведёт к узлу «Сад за замком»
3. **Уйти прочь** → ведёт к узлу «Конец игры»

```json
{
  "id": "entrance",
  "text": "Вы стоите перед старым замком...",
  "image": "castle.jpg",
  "choices": [
    { "text": "Войти", "target": "inside" },
    { "text": "Обойти", "target": "garden" },
    { "text": "Уйти", "target": "end" }
  ]
}
```

*Пример структуры узла в формате .mnd*

## Шаг 4: Добавление аудио

Фоновая музыка создаёт атмосферу. Поддерживаются форматы **MP3**, **OGG** и **WAV**.

Настройте громкость и зацикливание трека в панели свойств узла.

## Шаг 5: Условия и переменные

Для более сложных сценариев используйте **переменные** и **условия**:

### Пример: ключ от замка

```
Узел: "У ворот"
  └─ Выбор: "Попробовать открыть"
     ├─ Если has_key == true → "Ворота открылись!"
     └─ Если has_key == false → "Ворота заперты. Нужен ключ."
```

### Доступные операторы

| Оператор | Описание | Пример |
|----------|----------|--------|
| `==` | Равно | `health == 100` |
| `!=` | Не равно | `key != found` |
| `>` | Больше | `level > 5` |
| `<` | Меньше | `time < 30` |
| `>=` | Больше или равно | `score >= 1000` |
| `<=` | Меньше или равно | `lives <= 1` |

## Шаг 6: Тестирование

Нажмите ▶ **Запустить** чтобы протестировать квест. Проверьте все ветки и концовки.

![Тестирование квеста](/images/мокап программы из гугл плей красивый(не реальный скрин) 4.jpg)

*Режим тестирования*

---

## Шаг 7: Публикация

Когда квест готов — опубликуйте его на маркете:

1. Нажмите **Опубликовать**
2. Заполните метаданные
3. Выберите категорию
4. Нажмите **Отправить**

> 💡 **Совет:** Добавьте качественную обложку и описание — это увеличит шансы на то, что ваш квест заметят.
